# Система управления записями пациентов

Система автоматизации учета пациентов, реализованная на Python с использованием графического фреймворка PyQt6. Проект построен на строгом соблюдении архитектурного паттерна MVC, принципов ООП и обеспечивает работу с XML-документами.

## Архитектура проекта

Проект разделен на три основных слоя в соответствии с паттерном Model-View-Controller: данные и бизнес-логика (model), графический интерфейс (view) и управление (controller).

---

### Основные сущности

* Спортсмен - класс `Athlete`
* менеджер базы данных - `AthleteModel`
* обработчик XML - `AthleteHandler`
* главное окно приложения - `MainWindow`
* компоненты таблицы и интерфейса - `BaseTable`, `PaginationPanel`
* контроллер приложения - `PaginationController`

### Реализованные поведения
* операция добавления новых спортсменов
* операция многокритериального поиска 
* операция удаления записей по фильтрам
* операция пагинации (постраничного вывода) данных
* операция импорта (SAX) и экспорта (DOM) базы данных в формат XML

### Описание классов

**Paginator**: Центральный узел управления (Слой Controller). Выступает посредником между пользовательским интерфейсом и хранилищем данных. Управляет состоянием пагинации.

#### Слой данных (Model)
* **Athlete**: Содержит данные, описывающие спортсмена: ФИО, вид спорта, позиция, состав, количество титулов и разряд.
* **AtheleteModel**: Инкапсулирует логику управления массивом записей в оперативной памяти. Реализует методы фильтрации, поиска и удаления.
* **AthleteHandler**: Отвечает за десериализацию данных. Реализует событийно-ориентированный потоковый SAX-парсер (`xml.sax.ContentHandler`) для высокоэффективного импорта XML-файлов. Сохранение данных (экспорт) реализовано с помощью DOM-парсера (`xml.dom.minidom`).

#### Слой представления (View)
* **MainWindow**: Главное окно приложения. Агрегирует меню, панели инструментов, главную таблицу и нижнюю панель пагинации. 
* **BaseTable / PaginationPanel**: Изолированные графические компоненты. `BaseTable` настраивает столбцы и растягивает их по ширине окна, а `PaginationPanel` формирует макет с кнопками навигации и полем `QSpinBox`.
* **AddDialog / SearchDialog / DeleteDialog / EditDialog**: Модальные окна (`QDialog`) и виджеты, предоставляющие формы для взаимодействия с пользователем. 

---

Пользователь системы может:
* просматривать каталог спортсменов с использованием постраничной навигации
* менять количество отображаемых записей на одной странице
* добавлять новые записи, заполняя необходимые поля через виджеты
* искать, редактировать и удалять записи
* сохранять текущий массив записей в XML-файл или загружать данные из стороннего XML-документа.

### Интерфейс (Сценарии демонстрации)

Система предлагает полноценный графический интерфейс (GUI). Ниже описаны основные сценарии взаимодействия:

**Сценарий 1: Загрузка данных и пагинация**
1. При запуске открывается главное окно с пустой таблицей.
2. В верхнем меню пользователь выбирает `Файл -> Открыть`. Через системный диалог выбирается файл.
3. Таблица мгновенно заполняется первой страницей записей (по умолчанию 10). На нижней панели отображается статус текущей страницы.
4. Пользователь вводит новое значение в поле `Стр:`. Контроллер автоматически пересчитывает лимиты и обновляет таблицу. Кнопками навигации (`>`, `<`) осуществляется переход между страницами.

**Сценарий 2: Добавление новой записи**
1. В верхнем меню выбирается `Записи -> Добавить` (или используется кнопка на панели инструментов).
2. Появляется модальное окно `AddDialog` с полями ввода.
3. При попытке сохранить пустую форму система блокирует действие и выводит `QMessageBox` с ошибкой валидации.
4. Пользователь корректно заполняет поля и нажимает "Сохранить". Окно закрывается, объект добавляется в репозиторий, а главная таблица обновляется.
